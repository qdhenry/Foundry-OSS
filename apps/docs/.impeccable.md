## Design Context

Extracted from the [Bespoke Agentics brand system](../../../BESPOKE_AGENTICS/BespokeAgentics-website/) and adapted for the Starlight docs framework.

### Users

Developers, DevOps engineers, and agency technical leads evaluating or self-hosting Foundry. They arrive from GitHub, search, or word-of-mouth. Context: focused research mode — scanning docs for architecture decisions, setup steps, and API reference. They expect fast, scannable, credible documentation.

### Brand Personality

**Precise. Confident. Unhurried.**

Foundry is a professional delivery platform, not a toy. The brand communicates institutional competence through restrained typography, generous spacing, and a single-hue blue palette. No playfulness, no gradients, no decorative noise.

### Aesthetic Direction

- **Theme**: Dark-first (matches BA marketing site and the Foundry app), with full light mode support via Starlight's `data-theme` toggle.
- **Palette**: Single-hue blue (`#60a5fa` dark / `#2563eb` light`) on deep navy surfaces (`#050509`). Neutrals are blue-tinted, not gray. No purple, no cyan, no multi-color accents.
- **Typography**: DM Sans 300/400/500/600/700/800 (body + headings), DM Mono 300/400/500 (labels, code, captions), Instrument Serif 400 (display, optional emphasis).
- **Anti-references**: Gradient text, glassmorphism, side-stripe borders, sparkline decorations, rounded-rectangle-with-shadow card grids. See the impeccable skill's `absolute_bans`.

### Design Principles

1. **Token-driven consistency** — Every color, spacing, radius, and shadow comes from the shared `foundry-tokens.css` file. No hardcoded values in component styles.
2. **Starlight-native** — Override Starlight via its `--sl-*` CSS variables in `foundry-theme.css`. Only create custom components when Starlight has no equivalent.
3. **Editorial typography** — DM Sans at weight 600 for headings, 400 for body. Readable, not heavy. DM Mono at 0.7rem, uppercase, 0.2em letter-spacing for structural labels (section markers, tags, captions).
4. **Blue-tinted borders** — All borders use translucent blue `rgba(59, 130, 246, 0.12)` in dark mode, never opaque grays. This is the single biggest vibe differentiator from the live bespokeagentics.ai site.
5. **Restraint over decoration** — Card hover = border-color shift only, no glow shadows, no transforms. CTA buttons are flat solid blue, not gradient. Transitions are 0.25–0.3s, simple easing.
6. **Signal, not noise** — The BA brand mark (concentric quarter-arcs) is the only decorative motif. Everything else earns its pixels through utility.

### Token Architecture

```
foundry-tokens.css     → Defines all --surface-*, --text-*, --border-*, --accent-*,
                          --pill-*, --stat-*, --cta-*, --hero-*, --logo-*, --shadow-*
                          tokens for both light and dark themes.

foundry-theme.css      → Maps Foundry tokens to Starlight's --sl-* variables.
                          Also defines .foundry-pill and .foundry-label utility classes.

Head.astro             → Loads Google Fonts (DM Sans, DM Mono, Instrument Serif).
```

### Font Stack

| Mono     | DM Mono          | 300, 400, 500 | Code, labels, captions, pills      |
